# Plan 01-01: Schema & API — Per-Device Threshold Fields

## Goal

Thêm 2 cột `min_value` và `max_value` vào bảng `devices`. Tạo endpoint `PATCH /api/v1/devices/{deviceId}/threshold` để farmer cập nhật ngưỡng. Backend log WARNING khi telemetry vượt ngưỡng.

## Context

- `Device.java` hiện tại không có trường `minValue`/`maxValue` — cần thêm vào entity
- `ddl-auto: update` đang được dùng, nên Hibernate sẽ tự thêm cột vào DB khi khởi động
- `DeviceController.java` có pattern `PATCH /{deviceId}` cho rename — dùng cùng pattern cho threshold
- `MqttReceiverService.java` nhận telemetry và forward cho observers — đây là nơi validate ngưỡng
- JSON fields dùng `snake_case` (Jackson `SNAKE_CASE` strategy) — request/response phải dùng `snake_case`

### Task 3: Thêm method updateThreshold vào DeviceService

<read_first>
- `backend/api/src/main/java/com/yoloFarm/api/service/DeviceService.java`
- `backend/api/src/main/java/com/yoloFarm/api/repository/DeviceRepository.java` (xem available methods)
</read_first>

<action>
Trong `DeviceService.java` (interface hoặc implementation — xem codebase để xác định pattern), thêm method:

```java
// Interface (nếu có interface riêng):
Device updateThreshold(UUID ownerId, UUID deviceId, Float minValue, Float maxValue);

// Implementation:
public Device updateThreshold(UUID ownerId, UUID deviceId, Float minValue, Float maxValue) {
    Device device = deviceRepository.findByIdAndFarmOwnerId(deviceId, ownerId)
        .orElseThrow(() -> new EntityNotFoundException("Device not found or access denied"));
    device.setMinValue(minValue);
    device.setMaxValue(maxValue);
    return deviceRepository.save(device);
}
```

Nếu không có `findByIdAndFarmOwnerId`, dùng pattern tương tự `assertDeviceOwnership` hiện có: tìm device bằng id, kiểm tra farm.owner.id == ownerId, throw nếu không match.
</action>

<acceptance_criteria>
- `DeviceService` (interface hoặc impl) chứa method `updateThreshold` với signature nhận `UUID ownerId, UUID deviceId, Float minValue, Float maxValue`
- Method gọi `deviceRepository.save(device)` sau khi set minValue/maxValue
- Method throw exception nếu device không thuộc owner
</acceptance_criteria>

### Task 4: Thêm endpoint PATCH /{deviceId}/threshold vào DeviceController

<read_first>
- `backend/api/src/main/java/com/yoloFarm/api/controller/DeviceController.java`
- `backend/api/src/main/java/com/yoloFarm/api/dto/request/DeviceThresholdRequest.java` (vừa tạo)
</read_first>

<action>
Trong `DeviceController.java`, thêm import và endpoint mới (sau `updateDeviceName`):

```java
import com.yoloFarm.api.dto.request.DeviceThresholdRequest;

// Trong class:
@PatchMapping("/{deviceId}/threshold")
public ResponseEntity<?> updateThreshold(
        @AuthenticationPrincipal User currentUser,
        @PathVariable("deviceId") UUID deviceId,
        @RequestBody DeviceThresholdRequest request) {
    Device updated = deviceService.updateThreshold(
        currentUser.getId(), deviceId,
        request.getMinValue(), request.getMaxValue()
    );
    return ResponseEntity.ok(Map.of(
        "message", "Threshold updated successfully",
        "device_id", updated.getId(),
        "min_value", updated.getMinValue(),
        "max_value", updated.getMaxValue()
    ));
}
```
</action>

<acceptance_criteria>
- `DeviceController.java` chứa `@PatchMapping("/{deviceId}/threshold")`
- Endpoint nhận `@RequestBody DeviceThresholdRequest`
- Endpoint trả về response có keys `min_value` và `max_value`
</acceptance_criteria>

### Task 5: Validate ngưỡng trong MqttReceiverService khi nhận telemetry

<read_first>

```java
// Sau khi có: Device device = ...; float value = ...;
Float minVal = device.getMinValue();
Float maxVal = device.getMaxValue();
if (minVal != null && value < minVal) {
    log.warn("MqttReceiver: Value {:.2f} from device {} is below minimum threshold {:.2f}",
        value, device.getId(), minVal);
}
if (maxVal != null && value > maxVal) {
    log.warn("MqttReceiver: Value {:.2f} from device {} exceeds maximum threshold {:.2f}",
        value, device.getId(), maxVal);
}
```

Chỉ log WARNING — KHÔNG block/drop message. Dữ liệu vẫn được forward đến observers bình thường.
</action>

<acceptance_criteria>
- `MqttReceiverService.java` chứa `device.getMinValue()` và `device.getMaxValue()`
- Có `log.warn(...)` cho trường hợp giá trị dưới `minValue`
- Có `log.warn(...)` cho trường hợp giá trị vượt `maxValue`
- Logic vẫn gọi `notifyObservers()` sau validation (không block)
