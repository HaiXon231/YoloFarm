# Plan 04-03: Frontend Deploy on Vercel

## 1. Goal
Triển khai SPA React Vite lên Vercel, đảm bảo kết nối đúng với backend API đang chạy trên HuggingFace Spaces.

## 2. Requirements Addressed
- **DEPLOY-03**: Frontend deploy thành công lên Vercel với env vars đúng (API URL, WS URL).
- **DEPLOY-04**: E2E smoke test hoạt động mượt mà.

## 3. Scope of Work
### 3.1. Cấu hình Vite & Frontend variables
- Hiện tại frontend đang dùng `api.ts` hoặc `axios` trỏ tới `/api` thông qua Vite proxy ở chế độ development.
- Ở production, cần sử dụng biến môi trường (ví dụ `VITE_API_URL` và `VITE_WS_URL`) để gọi trực tiếp tới backend HuggingFace. Cần update `src/lib/axios.ts` và `src/lib/websocket.ts`.

### 3.2. Cấu hình Vercel
- Cấu hình `vercel.json` (nếu cần) để xử lý client-side routing (SPAs redirect mọi request về `index.html`). 
- Tuy nhiên, Vercel framework preset Vite đã tự động xử lý. Ta chỉ cần import project trên Vercel.
- Gán biến `VITE_API_URL` bằng HTTPS URL của HuggingFace backend (ví dụ `https://username-spacename.hf.space/api`).
- Gán biến `VITE_WS_URL` bằng WSS URL tương ứng.

### 3.3. E2E Smoke Test
- Người dùng truy cập Vercel link.
- Thực hiện Login thành công.
- Vào màn hình Farm, chờ WebSocket kết nối.
- Thử bật/tắt thiết bị để verify e2e flow.

## 4. Acceptance Criteria
- [ ] Vercel deploy thành công và truy cập được qua web.
- [ ] Giao diện có thể login, hiển thị farm, hiển thị dữ liệu realtime.

## 5. Potential Risks & Mitigations
- *Risk*: CORS Policy từ HuggingFace chặn request từ domain của Vercel.
- *Mitigation*: Tại backend (Spring Boot), phải cập nhật cấu hình CORS (`WebMvcConfigurer`) cho phép request từ domain Vercel (hoặc allow all `*` trong quá trình test).
