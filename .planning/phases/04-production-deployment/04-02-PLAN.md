# Plan 04-02: Backend Docker & HuggingFace Deploy

## 1. Goal
Đóng gói Spring Boot backend thành Docker container và triển khai lên HuggingFace Spaces.

## 2. Requirements Addressed
- **DEPLOY-01**: Dockerfile multi-stage build cho Spring Boot backend (Java 25).
- **DEPLOY-02**: Backend deploy thành công lên HuggingFace Spaces (port 7860).
- **DEPLOY-04**: Environment variables production được cấu hình đầy đủ.

## 3. Scope of Work
### 3.1. Viết Dockerfile
- Tạo `Dockerfile` trong thư mục `backend/api/`.
- Sử dụng Multi-stage build: 
  - Stage 1: Build với Maven (image `maven:3.9.6-eclipse-temurin-21` hoặc bản tương đương, lưu ý dùng Java tương thích với project — hiện tại project dùng Java 21, mặc dù yêu cầu ghi 25 nhưng temurin 25 chưa phổ biến, sẽ chốt Java 21).
  - Stage 2: Run với JRE (expose port 7860 theo chuẩn HuggingFace).
- Thêm file `.dockerignore` để bỏ qua `target/`, `.m2/`, vân vân.

### 3.2. Cấu hình HuggingFace Spaces
- Hướng dẫn người dùng tạo HuggingFace Space với cấu hình Docker.
- Cung cấp danh sách các Environment Variables cần thiết (Secrets):
  - `DB_URL`, `DB_USERNAME`, `DB_PASSWORD` (từ Neon.tech).
  - `ADAFRUIT_IO_KEY`
  - `JWT_SECRET_KEY`
  - `SPRING_PROFILES_ACTIVE=prod` (nếu có application-prod.yml).

### 3.3. Update application.yml cho Prod (Optional)
- Đảm bảo `application.yml` có thể nhận port từ biến môi trường `PORT` (HuggingFace set `PORT=7860` hoặc ta tự set `server.port: 7860`).

## 4. Acceptance Criteria
- [ ] Dockerfile build thành công local.
- [ ] Backend khởi động trên HuggingFace Spaces và API trả về HTTP 200 cho `/api/v1/auth/login`.

## 5. Potential Risks & Mitigations
- *Risk*: Không khớp phiên bản Java giữa local và Docker image.
- *Mitigation*: Set explicitly image version là Java 21 (hoặc version pom.xml đang dùng).
