# Plan 04-01: Database Deployment (Neon.tech)

## 1. Goal
Triển khai cơ sở dữ liệu PostgreSQL cho môi trường Production sử dụng Neon.tech, và đảm bảo schema được khởi tạo chính xác thông qua Flyway migration.

## 2. Requirements Addressed
- **DB-03**: Neon.tech PostgreSQL được cấu hình và kết nối thành công từ production backend.

## 3. Scope of Work
- Do đây là quá trình manual setup trên nền tảng cloud của người dùng, GSD agent sẽ cung cấp hướng dẫn (instructions) để người dùng thực hiện:
  1. Đăng ký/Đăng nhập Neon.tech.
  2. Tạo project mới "YoloFarm".
  3. Lấy Connection String (JDBC URL).
- (Tuỳ chọn) Chạy `mvnw flyway:migrate` trỏ tới Neon DB hoặc để ứng dụng Spring Boot tự động chạy Flyway khi khởi động lần đầu trên cloud.
- Cập nhật `.env` / thông số cấu hình Production cho backend (`DB_URL`, `DB_USERNAME`, `DB_PASSWORD`).

## 4. Acceptance Criteria
- [ ] Lấy được connection URL của Neon DB.
- [ ] Connect thử từ tool ngoài (DBeaver / DataGrip) thành công.
- [ ] Flyway áp dụng schema (hoặc sẽ áp dụng tự động ở bước 04-02).

## 5. Potential Risks & Mitigations
- *Risk*: Neon DB free tier có thể bị scale to zero gây cold start chậm hoặc ngắt kết nối.
- *Mitigation*: HikariCP trong Spring Boot thường tự retry, nhưng ở plan 04-02 cần cấu hình time-to-live hợp lý nếu cần.
