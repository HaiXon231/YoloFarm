# Plan 03-01: Test Setup & Store Tests

## 1. Goal
Thiết lập môi trường kiểm thử cho frontend React sử dụng Vitest và React Testing Library (RTL). Viết Unit Test cho các Zustand stores quan trọng nhằm đảm bảo trạng thái ứng dụng được quản lý chuẩn xác.

## 2. Requirements Addressed
- **TEST-03**: Frontend unit test cho các store Zustand (`authStore`, `notificationStore`)

## 3. Scope of Work
### 3.1. Install & Configure Vitest
- Cài đặt các package: `vitest`, `@testing-library/react`, `@testing-library/jest-dom`, `@testing-library/user-event`, `jsdom`.
- Tạo hoặc cập nhật `vite.config.ts` để thêm cấu hình test environment là `jsdom`, setupFiles cho RTL.
- Tạo `src/setupTests.ts` để import `@testing-library/jest-dom`.

### 3.2. Test `authStore.ts`
- Tạo file `src/stores/__tests__/authStore.test.ts`.
- Mock localStorage và test các action: `setCredentials`, `logout`.
- Kiểm tra tính đúng đắn của việc lưu token và thông tin user.

### 3.3. Test `notificationStore.ts`
- Tạo file `src/stores/__tests__/notificationStore.test.ts`.
- Test các thao tác: thêm thông báo (từ WebSocket mock), `markAsRead`, xóa thông báo, lấy số lượng chưa đọc.

## 4. Acceptance Criteria
- [ ] Chạy `npm run test` (hoặc `npx vitest`) thành công.
- [ ] Test của `authStore` coverage 100% các actions.
- [ ] Test của `notificationStore` coverage 100% các actions.
- [ ] Không có memory leak hoặc state bleed giữa các test case (cần reset zustand state trong `afterEach`).

## 5. Potential Risks & Mitigations
- *Risk*: Zustand persistence (localStorage) gây nhiễu state giữa các test case.
- *Mitigation*: Viết helper function để mock localStorage hoặc reset persist state của Zustand trước mỗi test run.
