# Plan 03-02: Page Integration Tests

## 1. Goal
Viết Component / Integration Test cho `FarmDetailPage` để đảm bảo luồng giao diện chính của hệ thống — bao gồm render danh sách cảm biến, biểu đồ telemetry và tương tác bật/tắt thiết bị — hoạt động ổn định và không bị break khi refactor.

## 2. Requirements Addressed
- **TEST-04**: Frontend integration test cho `FarmDetailPage` (hiển thị telemetry + điều khiển thiết bị).

## 3. Scope of Work
### 3.1. Mocking API / Services
- Sử dụng `vi.mock` (Vitest) để mock các hàm gọi API trong thư mục `src/services/` (ví dụ `farmService`, `deviceService`, `telemetryService`).
- Mock WebSocket (StompClient) nếu trang FarmDetail đăng ký hook `useWebSocket`.

### 3.2. Viết Test cho FarmDetailPage
- Tạo file `src/pages/__tests__/FarmDetailPage.test.tsx`.
- Cần wrap component trong `BrowserRouter` (do dùng `useParams`) và một custom Render provider nếu cần.
- Test case 1: Render Loading State khi đang fetch data.
- Test case 2: Component render thành công thông tin Farm, danh sách Devices và Chart.
- Test case 3: Hành vi tương tác (Click). Giả lập click nút toggle "ON/OFF" trên một Actuator Card, đảm bảo hàm gọi API `sendControlCommand` được trigger với đúng deviceId và command.

## 4. Acceptance Criteria
- [ ] Render thành công component trong môi trường jsdom.
- [ ] Chạy qua tất cả assertion verify hiển thị đúng dữ liệu mock.
- [ ] Bắt được event click và verify hàm mock API được gọi thành công.
- [ ] Coverage của file `FarmDetailPage.tsx` hoặc các hooks liên quan được tăng lên.

## 5. Potential Risks & Mitigations
- *Risk*: `FarmDetailPage` phụ thuộc vào Recharts, ResizeObserver (thường lỗi trên jsdom).
- *Mitigation*: Mock thư viện Recharts (trả về một stub div) hoặc cấu hình mock `window.ResizeObserver` trong `setupTests.ts` để test không bị crash.
