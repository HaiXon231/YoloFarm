# Plan 02-03: Backend Tests

## Goal
Đảm bảo hai luồng tự động quan trọng nhất là Rule Engine và Safety Watchdog hoạt động đúng qua integration test.

## Context
- `TEST-01`: Backend integration test cho Rule Engine pipeline (sensor data -> rule trigger -> actuator command).
- `TEST-02`: Backend integration test cho AutoIrrigationSafetyService (watchdog tự tắt sau `maxAutoOnMinutes`).

## Tasks

### Task 1: Integration Test cho Rule Engine (TEST-01)
<action>
Tạo file `backend/api/src/test/java/com/yoloFarm/api/service/automation/RuleEngineIntegrationTest.java`:
- Sử dụng `@SpringBootTest` (hoặc test slice nếu được) để verify luồng: Inject dữ liệu giả vào `RuleEngineService`, kiểm tra xem nó có gọi `ActionService` để gửi lệnh ON/OFF chính xác theo cấu hình Rules hay không.
- Mock các phụ thuộc bên ngoài (như MQTT/AdafruitApiService).
</action>

### Task 2: Integration Test cho Safety Watchdog (TEST-02)
<action>
Tạo file `backend/api/src/test/java/com/yoloFarm/api/service/automation/AutoIrrigationSafetyServiceTest.java`:
- Test các kịch bản:
  - Thiết bị vừa được bật ở chế độ AUTO -> Chưa vượt `maxAutoOnMinutes` -> Không làm gì.
  - Đã vượt quá `maxAutoOnMinutes` -> Watchdog tự động trigger tắt (gọi `ActionService.executeAction(OFF)`).
  - Tích hợp với `AutomationRuntimeStateService` để kiểm soát state.
</action>
