# Plan 02-02: Security Fixes

## Goal
Triển khai Rate Limiting cho login API để chống brute-force, tăng cường an toàn luồng (thread-safety) cho danh sách observer của MQTT, xử lý cache eviction khi rename thiết bị, và bắt lỗi quota từ Adafruit rõ ràng hơn.

## Context
- `SEC-01`: Đang thiếu Rate Limit cho `/api/v1/auth/login`.
- `SEC-02`: Đổi tên thiết bị ở `DeviceService.updateDeviceName()` quên chưa gọi `evictFeedKeyCacheSafe()`.
- `SEC-03`: Adafruit free tier giới hạn 10 feeds. Khi vượt, API ném ra lỗi 422 Unprocessable Entity. Cần bắt lỗi này và quăng exception tiếng Việt thân thiện để farmer biết.
- `SEC-04`: `MqttReceiverService` đang dùng `new ArrayList<>()` cho `observers` có rủi ro `ConcurrentModificationException` khi nhiều event đến đồng thời.

## Tasks

### Task 1: Rate Limiting với Bucket4j (SEC-01)
<action>
- Thêm `bucket4j-core` vào `pom.xml`.
- Tạo `RateLimitFilter` (cấu hình ví dụ 5 requests / phút / IP) chỉ áp dụng cho `/api/v1/auth/login`.
- Đăng ký filter này trong `SecurityConfig` (hoặc `FilterRegistrationBean`).
</action>

### Task 2: MQTT Thread-Safety (SEC-04)
<action>
Trong `backend/api/src/main/java/com/yoloFarm/api/service/mqtt/MqttReceiverService.java`:
- Thay `private final List<Observer> observers = new ArrayList<>();` thành `private final List<Observer> observers = new java.util.concurrent.CopyOnWriteArrayList<>();`.
</action>

### Task 3: Cache Eviction khi Rename (SEC-02)
<action>
Trong `backend/api/src/main/java/com/yoloFarm/api/service/DeviceService.java`, phương thức `updateDeviceName`:
- Gọi `evictFeedKeyCacheSafe(device.getAdafruitFeedKey())` sau khi đổi tên feed Adafruit thành công để đảm bảo lần gửi tiếp theo MQTT Receiver sẽ query tên feed chuẩn nhất.
</action>

### Task 4: Adafruit Quota Guard (SEC-03)
<action>
Trong `backend/api/src/main/java/com/yoloFarm/api/service/AdafruitApiService.java`:
- Trong phương thức `createFeed`, chặn lỗi HTTP 422 (chứng tỏ vượt quá quota 10 feeds). Ném ra `ConflictException("Tài khoản Adafruit đã đạt giới hạn feed (tối đa 10). Không thể tạo thêm thiết bị.")`.
</action>

## Verification
- Biên dịch thành công.
- Không còn cảnh báo concurrency.
