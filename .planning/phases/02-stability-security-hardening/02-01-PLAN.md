# Plan 02-01: Flyway Migration — Baseline V1

## Goal
Thay thế cơ chế tự động tạo bảng của Hibernate (`ddl-auto: update`) bằng Flyway migration để quản lý schema an toàn hơn trên production. Tạo script migration V1 làm baseline.

## Context
- Ứng dụng đang dùng `spring.jpa.hibernate.ddl-auto: update`. Rất rủi ro khi đưa lên production vì có thể gây xóa hoặc thay đổi dữ liệu ngoài ý muốn.
- Cần thêm dependency `flyway-core` (và `flyway-database-postgresql` nếu dùng Flyway 10+).
- Cần tạo file `V1__init_schema.sql` chứa cấu trúc bảng hiện tại (bao gồm cả các cột `min_value`, `max_value` vừa thêm ở Phase 1).

## Tasks

### Task 1: Thêm Flyway dependency
<read_first>
- `backend/api/pom.xml`
</read_first>

<action>
Thêm dependency `flyway-core` (và `flyway-database-postgresql`) vào `pom.xml`.
</action>

### Task 2: Tạo Flyway baseline script
<read_first>
- Kiểm tra các Entity trong `backend/api/src/main/java/com/yoloFarm/api/entity/` để lấy cấu trúc cột chính xác.
</read_first>

<action>
Tạo file `backend/api/src/main/resources/db/migration/V1__init_schema.sql` chứa toàn bộ câu lệnh CREATE TABLE cho hệ thống (users, farms, models, devices, rules, notifications, v.v.). Đảm bảo bao gồm `min_value` và `max_value` trong bảng `devices`.
</action>

### Task 3: Cấu hình application.yml
<read_first>
- `backend/api/src/main/resources/application.yml`
</read_first>

<action>
Đổi cấu hình Hibernate:
- `spring.jpa.hibernate.ddl-auto: validate`
- Thêm `spring.flyway.enabled: true` và `spring.flyway.baseline-on-migrate: true` (hoặc cấu hình cần thiết để Flyway chạy thành công trên DB có sẵn/mới).
</action>

## Verification
- Chạy ứng dụng (`mvnw spring-boot:run` hoặc `compile`), Flyway sẽ apply V1 script thành công.
- Không còn lỗi ddl-auto.
